JS 预编译，执行前的最后疯狂~

Js 运行三部曲

一、JS 运行三部曲

语法分析→预编译→解释执行

语法分析是扫描，寻找语法错误，但是不执行。

补充知识点：var a = function () {} 这个是函数表达式；function a() {} 这个是函数声明

二、预编译前奏

函数声明整体提升，变量声明提升。

全局变量特征(window 就是全局的域)：

  • 任何变量，如果未经申明直接赋值，那么就是全局变量,，即是 window 所有。比如

    function （）{var a = b = 30;}

    此时 b 就是全局变量。a 是局部变量。

  • 一切定义在全局上的变量,都是全局变量。都是 window 的属性。

    var a = 3; 就相当于 window.a = 39;

    console.log(a) 就相当于 console.log(window.a);

三、预编译过程

预编译发生在函数执行的前一刻。

预编译步骤：4 部曲

  1. 创建 AO 对象（activition object）（函数执行时的上下文）

  2. 找形参和变量声明（即变量声明提升，此时函数声明也整体提升），将变量和形参名作为 AO 属性名，值为 undefined.

  3. 将实参值和形参统一

  4. 在函数体里面找函数声明，函数的名字作为对象的属性名，函数体作为属性的值。

在全局是 GO 对象，GO 就是 window.
